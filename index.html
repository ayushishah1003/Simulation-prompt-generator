What Are Prompt Frameworks?

How Frameworks Enhance Your Simulation Prompt

A prompt framework is a structured instructional approach embedded directly into your Claude prompt. It tells Claude not just what to build, but how students should cognitively engage with it — changing the pedagogical architecture of the simulation itself.

Think of it this way: Without a framework, Claude builds a simulation. With Chain of Thought, Claude builds a simulation where students must verbalise their reasoning before every decision. With Tree of Thought, students evaluate three possible futures before the real one is revealed. The same topic, radically different learning experiences.
All 9 Frameworks

Framework Reference — What Each One Does

RTF
Relevant · Transparent · Feedback
Grounds the simulation in student prior knowledge, makes learning objectives explicitly visible at the start, and builds structured feedback loops into every round outcome.
What it adds to your prompt:
  • Learning objectives banner at simulation start
  • Each variable labelled with the prior concept it connects to
  • Structured "What did you learn?" prompt after every result
  • Feedback framed as: "Your decision reflects… This connects to…"
SPARK
Specific · Purposeful · Adaptive · Relevant · Knowledge-building
Ensures every simulation element has a declared purpose and that difficulty adapts to how students are performing — preventing frustration or boredom.
What it adds:
  • Purpose tooltip on each slider ("This slider models…")
  • Mid-simulation nudge if student is repeatedly incorrect
  • Knowledge-building: each round recap builds into the next
  • Adaptive hint system triggered by poor prediction accuracy
CREATE
Creative · Relevant · Engaging · Authentic · Thoughtful · Exploratory
Opens the simulation beyond preset scenarios — students can propose their own variable values, suggest alternative outcomes, and design their own "what if" extension.
What it adds:
  • "Design your own scenario" mode after round 3
  • Student can propose a new slider variable mid-game
  • Creative challenge: "Redesign this system to avoid collapse"
  • Open exploration phase with no locked prediction required
GRADE
Goals · Rigor · Audience · Differentiation · Evidence
Assessment-focused framework. Aligns each round to a specific IB criterion, provides three differentiated entry points, and generates evidence students can submit.
What it adds:
  • IB criterion badge on each round (e.g. "Criterion B: Investigating")
  • Three difficulty tracks: Standard / Challenge / Extended HL
  • Evidence log panel — students can export decisions + outcomes
  • Self-assessment rubric at simulation end
Persona Prompting
Role-based Expert Framing
Claude adopts the persona of a domain expert throughout the simulation — responses, advisor tips, and result explanations all come through the expert's voice and knowledge.
What it adds:
  • Claude introduced as: "You are Dr. [Name], a veteran [specialist]…"
  • All advisor chips speak in-character as domain experts
  • Results explained: "As a marine biologist, I would note that…"
  • Expert language calibrated to the curriculum level
Chain of Thought (CoT)
Step-by-Step Reasoning Externalisation
Before advancing each round, students must complete a reasoning chain in writing — "I chose X because… which will lead to… because… therefore I predict…" — making thinking visible.
What it adds:
  • Mandatory reasoning text box before Advance activates
  • Advance button stays DISABLED until all chain blanks filled
  • After reveal: Claude evaluates the reasoning chain quality
  • Highlights logical gaps: "Step 2 of your chain assumed X, but…"
Tree of Thought (ToT)
Multi-Path Futures Evaluation
Before the real result is revealed, three plausible future scenarios are shown based on the student's decisions. Students rank them by likelihood with reasoning — then the real outcome appears.
What it adds:
  • Three "possible futures" cards shown before reveal
  • Students rank futures 1–3 by probability and briefly justify
  • Real outcome revealed with: "You chose [rank]. The actual outcome was…"
  • Discussion prompt: "Why did Future B happen instead of A?"
ReAct
Reason · Act · Observe · Reflect
Structures every round as a four-stage explicit cycle — students move through labelled RAOR stages, keeping cognitive process visible and structured throughout.
What it adds:
  • Round UI split into 4 labelled panels: Reason / Act / Observe / Reflect
  • REASON: Student writes hypothesis before touching sliders
  • OBSERVE: Results shown with structured reading guide
  • REFLECT: "How does this change your mental model?" — required field
Self-Consistency (SC)
Multi-Perspective Verification
After the student makes a prediction, three advisor characters each make their own prediction with brief reasoning — visible to the student. Students compare reasoning before the reveal.
What it adds:
  • 3 advisor predictions with 1-sentence reasoning each
  • Student sees: "Advisor A predicts Thriving because… Do you agree?"
  • Post-reveal: "Which advisor's reasoning was most accurate? Why?"
  • Metacognitive prompt: "Did any advisor change your thinking?"

Power Tips

Getting Better Results Every Time

Be specific about visual feedback

Say "when forest drops below 30%, trees shrink and turn brown." Claude executes what you describe literally — vague requests produce vague outputs.

Build in genuine trade-offs

The best simulations have decisions that improve one metric while hurting another. Explicitly state: "high farming = more food but less water."

Layer frameworks for depth

Chain of Thought + ReAct = students reason AND cycle. Tree of Thought + Self-Consistency = collaborative reasoning before reveal. The combinations create entirely different experiences.

Use the Prompt Modifier

After generating, click any modifier card to send targeted follow-ups to Claude — retheme, add HL questions, generate a worksheet — without rewriting the full prompt.

For assessments: use GRADE framework

GRADE embeds IB criterion alignment, three difficulty tracks, and an evidence log — making the simulation usable as a formal summative task with minimal extra setup.

For DP: add Persona Prompting

Having Claude adopt a domain expert persona calibrates the language, explanations, and advisor tips to the rigour expected in DP subjects — especially useful for Sciences and Economics.

Before Class

Quality Checklist

  • Open HTML in Chrome or Firefox (SVG animations perform best in Chrome)
  • Advance through all rounds — check no JavaScript errors (F12 → Console tab)
  • Verify prediction widget disables Advance button until prediction is selected
  • If using Chain of Thought or ReAct: verify reasoning boxes appear and are required
  • If using Tree of Thought: verify three futures appear before the reveal card
  • Check resource bars animate and shift colour green → amber → red correctly
  • Confirm IB Reflection panel opens with your custom discussion questions
  • Verify the visual scene changes meaningfully at slider extremes
  • Test Reset button restores all starting values
  • Disconnect WiFi and reload — should work offline (only Google Fonts need internet)

Assessment Security

Keeping Assessment Simulations Private

The problem: If a student follows your GitHub profile, they can see all public repositories — including assessments not yet released. Four solutions below.
A

Unpublish Pages (simplest)

Keep the repo Public but unpublish Pages until assessment day. Go to Settings → Pages → Source → None. The URL stops working. Re-enable on assessment day — live again in ~60 seconds.

B

Private repository

Create the repo as Private. GitHub Pages on private repos requires a paid plan. Workaround: keep private until assessment day, then switch to Public + enable Pages, then switch back.

C

Separate assessment account (recommended)

Create a second GitHub account (e.g. fountainhead-assessments) that students don't follow. Upload all assessments there. Share the URL only at assessment time.

D

Google Drive (no GitHub needed)

Upload HTML to Google Drive. Share → "Anyone with the link can view". Opens directly in Chrome. Share link only at assessment time via ManageBac.

Team recommendation: Use Option A for teaching simulations. Use Option C for formal assessments. Screen-recording the simulation before releasing the URL works well to preview it with students.
