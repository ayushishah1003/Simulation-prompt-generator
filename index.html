Power Tips

Getting Better Results Every Time

🎯

Be specific about visual feedback

Don't say "make it interactive." Say "when forest drops below 30%, trees shrink and turn brown." Claude executes what you describe literally.

⚖️

Build in genuine trade-offs

The best simulations have decisions that improve one metric while hurting another. Explicitly state the tension: "high farming = more food but less water."

🔄

Iterate in the same chat

After Claude generates the file, say "change the colour scheme" or "add a sixth slider for X" — it regenerates with just that change, keeping everything else.

📱

Request a simplified version

For Grade 6 or EAL learners, Claude will offer this automatically after generating (the AFTER GENERATING section handles it). Just pick option A.

🌐

Add a news ticker

Append to your prompt: "Include a scrolling news ticker at the top displaying headlines matching current simulation conditions." Grounds the simulation in urgency.

📊

Ask for examiner questions

For DP subjects, add: "After each round, display an IB Paper 1 / Paper 2 style question (with mark allocation) related to the outcome, before revealing results."

Before Class

Quality Checklist

  • Open HTML in Chrome or Firefox (not Safari — SVG animations perform best in Chrome)
  • Advance through all rounds to confirm no JavaScript errors (check browser console: F12)
  • Verify prediction widget disables Advance button until a prediction is selected
  • Check resource bars animate smoothly and change colour green → amber → red
  • Confirm the IB Reflection panel opens with your custom discussion questions
  • Verify the visual scene changes meaningfully when you drag sliders to extremes
  • Confirm at least one random event triggers mid-game
  • Test the Reset button restores all starting values
  • Disconnect WiFi and reload — should work offline (only Google Fonts need internet)

1

Create a free GitHub account

Go to github.com and sign up with your school email. Use your teacher name, e.g. ms-shah-fountainhead. Verify your email before continuing.

2

Create a new repository for your simulation

Click the green New button. Name it something descriptive, e.g. ib-eco-market-sim. Choose:

  • Public — anyone with the link can see it (good for teaching)
  • Private — only you can see it (use for assessment simulations before release)

Tick "Add a README file" and click Create repository.

3

Upload your HTML simulation file

Click Add file → Upload files. Drag your HTML file in. Name it index.html (this is important — GitHub Pages looks for this name). Click Commit changes.

4

Enable GitHub Pages

Go to Settings → Pages (left sidebar). Under Source, select Deploy from a branch, then choose main branch and / (root). Click Save.

Wait 60–90 seconds. Refresh the Settings → Pages page. Your URL will appear in a green box: Your site is live at https://...

5

Share the link with students

Copy the URL from Settings → Pages. Paste it into ManageBac, Google Classroom, or your class chat. Students click it — the simulation opens immediately in their browser with no download or login needed.

Assessment Security

Keeping Assessment Simulations Private

This was raised in your team meeting — here are the solutions, from simplest to most secure.

⚠️ The problem If a student follows your GitHub profile, they can see all your public repositories — including assessment simulations you haven't released yet. Here are four ways to solve this.
A

Option A — Unpublish Pages (simplest)

Keep the repository Public but unpublish the Pages site until the assessment day. Go to Settings → Pages → change Source to None. The URL stops working. On assessment day, re-enable Pages — the URL is live again in ~60 seconds. Students who bookmarked the link get a 404 until you re-enable it.

B

Option B — Private repository (recommended for assessments)

Create the repository as Private. Upload and test your simulation. GitHub Pages works on private repos only on paid GitHub plans. The workaround: keep it private until the day, then change to Public + enable Pages for the assessment window, then switch back to Private.

C

Option C — Use a separate "assessment" GitHub account

Create a second GitHub account (e.g. fountainhead-assessments) that students don't follow. Upload all assessment simulations there. No student can discover it unless you share the direct URL. Share the URL only at the moment of the assessment.

D

Option D — Google Drive (no GitHub needed)

Upload the HTML file to Google Drive. Right-click → Share → "Anyone with the link can view". The file opens directly in Chrome. For assessments: share the link only at assessment time via ManageBac. Previous simulations stay in Drive but aren't accessible without the direct link.

💡 Team meeting recommendation (Varsil Sir to confirm) Use Option A (unpublish/publish Pages) for teaching simulations. Use Option C (separate assessment account) for all formal assessments. This gives you the fastest workflow with the cleanest security boundary. Screen-recording workflow (mentioned in your meeting) works well as a supplementary approach for showing students what the simulation looks like before they can access it.
Updating Your Simulation

How to Update a Live Simulation

1

Ask Claude to make changes

In the same Claude conversation, say: "Change the colour scheme to darker", "Add a sixth slider for X", or "Fix the prediction button". Claude regenerates the complete HTML.

2

Re-upload to GitHub

Go to your repository. Click on index.html. Click the pencil (edit) icon. Delete all content. Paste your new HTML. Click Commit changes. The live URL updates within 30–60 seconds.

3

Students hard-refresh

Students press Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac) to force-reload the latest version. No app update, no reinstall — it's instant.
