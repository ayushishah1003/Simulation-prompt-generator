What Are Prompt Frameworks?

How Frameworks Enhance Your Simulation

A prompt framework is a structured instructional approach embedded into the Claude prompt. It tells Claude not just what to build, but how students should cognitively engage — changing the pedagogical architecture of the simulation itself.

Think of it this way: Without a framework, Claude builds a simulation. With Chain of Thought, students must verbalise their reasoning before every decision. With Tree of Thought, they evaluate three possible futures before the real one is revealed. Same topic — radically different learning experiences.
All 9 Frameworks

Framework Reference

RTF
Relevant · Transparent · Feedback
Grounds the simulation in student prior knowledge, makes learning objectives explicitly visible at the start, and builds structured feedback loops into every round outcome.
What it adds:
  • Learning objectives banner at simulation start
  • Each variable labelled with prior curriculum link
  • Structured feedback box after every result
  • Feedback framed as: Your decision reflects… This connects to…
SPARK
Specific · Purposeful · Adaptive · Relevant · Knowledge-building
Ensures every simulation element has a declared purpose and that difficulty adapts to student performance — preventing frustration or boredom mid-simulation.
What it adds:
  • Purpose tooltip on each slider: This slider models…
  • Mid-simulation nudge if student is repeatedly incorrect
  • Knowledge-building: each round recap builds into next
  • Adaptive hint system triggered by poor prediction accuracy
CREATE
Creative · Relevant · Engaging · Authentic · Thoughtful · Exploratory
Opens the simulation beyond preset scenarios — students can propose their own variable values, suggest alternative outcomes, and design a what-if extension.
What it adds:
  • Design-your-own scenario mode after round 3
  • Student can propose a new slider variable mid-game
  • Creative challenge: Redesign this system to avoid collapse
  • Open exploration phase with no locked prediction required
GRADE
Goals · Rigor · Audience · Differentiation · Evidence
Assessment-focused. Aligns each round to a specific IB criterion, provides three differentiated entry points, and generates evidence students can export for submission.
What it adds:
  • IB criterion badge on each round
  • Three difficulty tracks: Standard / Challenge / Extended HL
  • Evidence log panel — export decisions and outcomes as CSV
  • Self-assessment rubric aligned to IB criteria at simulation end
Persona Prompting
Role-based Expert Framing
Claude adopts the persona of a domain expert throughout the simulation — responses, advisor tips, and result explanations all come through the expert's voice and knowledge base.
What it adds:
  • Claude introduced as a named expert in the field
  • All advisor chips speak in-character as domain specialists
  • Results explained from expert perspective with specific data
  • Expert language calibrated to the IB level and ATL skills
Chain of Thought (CoT)
Step-by-Step Reasoning Externalisation
Before advancing each round, students must complete a four-part reasoning chain in writing — making thinking visible and surfacing logical gaps before the result is revealed.
What it adds:
  • Mandatory reasoning text box before Advance activates
  • All four chain steps required before Advance unlocks
  • Post-reveal: Claude evaluates the reasoning chain quality
  • Highlights the exact step where logical gaps occurred
Tree of Thought (ToT)
Multi-Path Futures Evaluation
Before the real result is revealed, three plausible future scenarios are shown based on the student's decisions. Students rank them by likelihood — then the actual outcome appears.
What it adds:
  • Three possible futures cards shown before reveal
  • Students rank futures 1-3 with brief justification
  • Real outcome revealed: You chose rank X. The actual outcome was…
  • Discussion prompt: Why did Future B happen instead of A?
ReAct
Reason · Act · Observe · Reflect
Structures every round as a four-stage explicit cycle — students move through labelled RAOR stages, keeping cognitive process visible and structured throughout.
What it adds:
  • Round UI split into 4 labelled panels: Reason / Act / Observe / Reflect
  • REASON: Student writes hypothesis before touching sliders
  • OBSERVE: Results with structured reading guide
  • REFLECT: How does this change your mental model? — required field
Self-Consistency (SC)
Multi-Perspective Verification
After the student makes a prediction, three advisors each make their own prediction with reasoning — visible before the reveal. Students compare their logic to expert views.
What it adds:
  • 3 advisor predictions with 1-sentence reasoning each
  • Student sees advisor reasoning before the reveal
  • Post-reveal: Whose reasoning was most accurate? Why?
  • Metacognitive prompt: Did any advisor change your thinking?

Power Tips

Getting Better Results Every Time

Be specific about visual feedback

Say "when forest drops below 30%, trees shrink and turn brown." Claude executes what you describe literally — vague requests produce vague outputs.

Build in genuine trade-offs

The best simulations have decisions that improve one metric while hurting another. Explicitly state: "high farming = more food but less water."

Layer frameworks for depth

Chain of Thought + ReAct = students reason AND cycle. Tree of Thought + Self-Consistency = collaborative reasoning before reveal. Combinations create entirely different learning experiences.

Use the Prompt Modifier

After generating, click any modifier card to send targeted follow-ups to Claude — retheme, add HL questions, generate a worksheet — without rewriting the full prompt.

For assessments: use GRADE framework

GRADE embeds IB criterion alignment, three difficulty tracks, and an evidence log — making the simulation usable as a formal summative task with minimal extra setup.

For DP: add Persona Prompting

Having Claude adopt a domain expert persona calibrates the language, explanations, and advisor tips to the rigour expected in DP Sciences, Economics, and Mathematics.

Before Class

Quality Checklist

  • Open HTML in Chrome or Firefox — SVG animations perform best in Chrome. Check on the projector resolution before class.
  • Advance through all rounds — check no JavaScript errors (press F12 then Console tab, look for red errors)
  • Verify prediction widget disables Advance button until a prediction is selected
  • If Chain of Thought or ReAct active: verify reasoning boxes appear and are required before Advance unlocks
  • If Tree of Thought active: verify three futures appear before the reveal card
  • If GRADE active: verify IB criterion badges appear on each round and evidence log is accessible
  • Check resource bars animate and shift colour correctly: green to amber to red
  • Confirm IB Reflection panel opens with your custom discussion questions
  • Verify the visual scene changes meaningfully when you drag sliders to extremes
  • Test Reset button — restores all starting values and clears student entries
  • Disconnect WiFi and reload — should work offline (only Google Fonts need internet)

Create a repository for your simulation

Click the green New button. Name it descriptively: e.g. ib-eco-island-sim or dp-chemistry-haber. Choose:

  • Public — anyone with the link can view (good for teaching simulations)
  • Private — only you can see it (use for assessments before release)

3

Upload your HTML simulation file

Click Add file → Upload files. Drag your HTML file in. Rename it index.html before uploading — this is critical. GitHub Pages serves index.html by default. Click Commit changes.

4

Enable GitHub Pages under your account

Go to your repository → Settings → Pages (left sidebar). Under Source, select Deploy from a branch, choose main and / (root). Click Save.

Wait 60–90 seconds. Refresh Settings → Pages. Your personal URL appears: https://[your-username].github.io/[repo-name]/

5

Share with students via ManageBac or Google Classroom

Copy the URL from Settings → Pages. Paste into ManageBac assignment, Google Classroom, or your class chat. Students click — the simulation opens immediately in their browser. No download, no login, no app.

Updating Your Simulation

How to Update a Live Simulation

1

Ask Claude to make the change

In the same Claude conversation, say: "Change the colour scheme to blue-slate" or "Add a sixth slider for X" or "Fix the prediction button." Claude regenerates the complete HTML.

2

Re-upload to your GitHub repository

Go to your repo. Click on index.html. Click the pencil (edit) icon. Delete all existing content. Paste your new HTML. Click Commit changes. The live URL updates within 30–60 seconds.

3

Students hard-refresh

Students press Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac) to force-reload the latest version. No reinstall, no new link needed.

Assessment Security

Keeping Assessment Simulations Private

Since the simulation URL is under your personal account, students who find your GitHub profile could see all your public repositories. Here are four solutions, from simplest to most secure.

The problem: If a student searches for your GitHub username, they can see all public repositories — including assessments you have not released yet. Use one of the options below.
A

Unpublish Pages (simplest — recommended for most cases)

Keep the repo Public but unpublish the Pages site until assessment day. Go to Settings → Pages → Source → None. The URL stops working. On assessment day, re-enable Pages — live again in about 60 seconds.

B

Private repository (best for high-stakes assessments)

Create the repo as Private. GitHub Pages on private repos requires a paid plan. Workaround: keep it private until assessment day, then switch to Public and enable Pages for the assessment window, then switch back to Private.

C

Separate assessment account (most secure)

Create a second GitHub account (e.g. fountainhead-assessments-shah) that students have never seen. Upload all formal assessment simulations to that account. Share the URL only at assessment time via ManageBac. Students cannot discover it without the direct link.

D

Google Drive (no GitHub needed)

Upload the HTML file to your personal Google Drive. Right-click → Share → "Anyone with the link can view". The file opens directly in Chrome. Share the link via ManageBac only at assessment time.

Recommended approach for Fountainhead: Use Option A (unpublish and republish Pages) for teaching simulations. Use Option C (separate assessment account) for all formal assessments. This gives you the fastest workflow with the cleanest security boundary — and keeps your personal teaching simulations easily accessible at all times.
